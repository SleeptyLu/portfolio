Porquê POO?

Escrever código é fácil; escrever código que outra pessoa (ou a "eu" do futuro) consiga ler e expandir é o verdadeiro desafio. A POO permite-me criar "moldes" (classes) para tudo o que existe no meu jogo de pesca.

Em vez de escrever código repetitivo para cada peixe, crio uma classe base Peixe com comportamentos comuns, e uso herança para criar variações únicas.

Sistemas de Itens

Todos os itens do inventário partilham uma classe base. Isto permite que o sistema de inventário trate uma cana de pesca e um peixe da mesma forma na hora de os guardar, mas de forma diferente na hora de os usar.

IA de Peixes

Através do polimorfismo, diferentes peixes podem ter métodos de "nadar" diferentes, mas o motor do jogo chama apenas a função peixe.move() sem precisar de saber que tipo de peixe é.

// Exemplo de Herança em GDScript/C#
